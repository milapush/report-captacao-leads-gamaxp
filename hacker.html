AUDITORIA

A auditoria da landing page Help Gestão foi realizada com a ferramenta PageSpeedInsights e foi obtida a nota 97 para a versão desktop e 80 para a versão mobile. Essa ferramenta apontou alguns pontos de melhoria conforme resumo abaixo:

Mobile

Desktop

Pontos importantes da auditoria:

  • Largest Contentful Paint elemento: Neste diagnóstico foi informado que o elemento "h1" foi o elemento de maior comprimento na tela, porém foi decidido manter, pois se trata de um título.

  • Avoid large layout shifts – O valor de CLS Contribution é de 0,012 dentro de uma escala de 0 a 1 foi considerado adequado.

  • Avoid long main-thread tasks- Diagnostica as tarefas mais longas no thread principal, que pode atrasar a entrada. Porém estas informações são os códigos de trackeamento utilizados pelo marketing.

  • Mantenha as contagens de solicitações baixas e os tamanhos de transferência pequenos: Disponibilize recursos estáticos com uma política de cache eficiente: Foi identificado o cache com curto ciclo de vida, mas a página será acessada uma vez por usuário não comprometendo sua experiência.

  • Remova o JavaScript não utilizado: É o código de trackeamento do marketing.

    Criar repositório no GitHub; Criar conta no Netlify; Conectar repositório ao Netlify;

    Esses itens foram primordiais para que o nosso site entrasse ar. Não houve dificuldade para realizar esses passos. Já tinhamos experiência com o GitHub e embora não conhecessemos o Netlify, a plataforma é super fácil de usar.

    Disponibilizar domínio; Apontar domínio para DNS do Netlify;

    A princípio não foi difícil disponibilizar o domínio do Netlify. Porém no decorrer da tarefa, descobrimos que era obrigatório ter um domínio próprio. A partir disso, compramos o domínio em outro site e não houve dificuldade em apontar o mesmo para DNS do Netlify. Entretanto, acabou rolando um certo "susto" pois a plataforma dizia que levava 2h para concluir o processo. O nosso levou quase 3h. E já estávamos tentando descobrir o que havíamos feito de errado. Porém não foi necessário alterar nada, o processo estava correto. Só demorou mais pra subir mesmo.

    Dar push no repositório com HTML básico; Ver o seu site disponível na web;

    Não houve dificuldades em executar essa tarefa. A Hacker responsável já tinha experiência com essa parte.

    Colocar formulário disponibilizado pela Gama em projeto;

    Era item necessário para o lead registrar seus dados. O formulário funcionou como planejado assim que implantado.

    Testar cadastro de leads no formulário;

    Não houve dificuldades em executar essa tarefa. Mantivemos o DebugMode durante todo o período de teste. Só retiramos para fazer 1 único teste em produção, que correu tudo certo.

    Criar página de destino após “submit” do formulário;

    Era um item necessário para a entrega porque era a página que a pessoa veria após colocar seus dados. A página sofreu mudanças no layout com o decorrer do tempo. Houve problema na construção da divisão da página, mas foi resolvido no final.

    Configurar página de destino como parâmetro no formulário;

    A tarefa em si não era difícil, pois era só incluir a url da página de destino no parâmetro value que estava no input de redirect. Entretanto, a Hacker responsável gostaria de fazer uma "rota" dentro do próprio site. Pesquisou diversas formas de fazer, porém teve medo que conflitasse com o back-end que já estava pronto. Então preferiu não fazer desta forma. E colocar somente a URL da Thank You Page. E para isso, tivemos que hospedar a Thank You page no Netlify também.

    Criar e iterar sob design enviado por hipsters;

    Foi estruturada uma página de exemplo e a partir do design enviado pelas hipsters houve uma reestruturação nos códigos. Não foi utilizado framework e por isso levou mais tempo. Porém no final foi entregue em tempo hábil.

    Garantir que o layout não está quebrando em versão mobile;

    Utilizamos a propriedade Grid CSS para estruturar a nossa página e acredito que isso facilitou na adaptação para o mobile. Na Thank You page, por ser somente 2 sessões, optamos por não fazer o grid e nessa sim, a responsividade foi um pouco mais demorada.

    Integrar pixels de rastreamento (Google, Facebook, etc) em páginas do projeto;

    Essa tarefa era essencial para que as Hypers conseguissem ter dados para analisar a página. Foi fácil de impletar pois os códigos já vieram prontos do Google, e a plataforma também já informava as tags corretas onde deveriam ser colocados.

    e boas práticas (web.dev) e aplicar correções necessárias;

    Foi realizada a auditoria com a ajuda de algumas ferramentas e as avaliações do site foram muito boas.
